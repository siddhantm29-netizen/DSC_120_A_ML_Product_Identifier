## Architecture

### Custom CNN — CustomResNetSE_CNN

A deep residual network built from scratch with Squeeze-and-Excitation (SE) attention:

```
Input (192×192×3)
  → Stem: Conv-BN-Swish ×2 (48, 64 filters)
  → Stage 1: Residual-SE ×2 (64 ch)
  → Stage 2: Residual-SE ×2 (128 ch, stride 2, dropout 3%)
  → Stage 3: Residual-SE ×3 (256 ch, stride 2, dropout 6%)
  → Stage 4: Residual-SE ×2 (384 ch, stride 2, dropout 8%)
  → Stage 5: Residual-SE ×2 (512 ch, stride 2, dropout 10%)
  → GAP → Dropout(0.35) → Dense(512)-BN-Swish → Dropout(0.35) → Softmax(10)
```

Each SE block: squeeze channels ÷8 → ReLU → expand → sigmoid gate.

**Training:** AdamW (lr=1e-3, cosine decay), weight decay λ=1e-4, MixUp α=0.20, label smoothing ε=0.05, 25 epochs.

### MobileNetV2 Transfer Learning

- **Backbone:** MobileNetV2 pre-trained on ImageNet (frozen)
- **Head:** GAP → Dropout(0.30) → Dense(10, softmax)
- **Phase 1:** Head only, 8 epochs, lr=1e-3
- **Phase 2:** Top 30% of layers unfrozen, 18 epochs, lr=1e-5 (BatchNorm stays frozen)

## Reproducing Results

### 1. Clone and set up

```bash
git clone https://github.com/siddhantm29-netizen/DSC_120_A_ML_Product_Identifier
cd fashion-classification
pip install tensorflow tensorflowjs kaggle
```

### 2. Download the dataset

```bash
kaggle datasets download paramaggarwal/fashion-product-images-small
unzip fashion-product-images-small.zip -d data/
```

### 3. Run training

Open `notebook2d94837274.ipynb` in Kaggle Notebooks or Jupyter. The notebook:
- Filters the dataset to 10 classes
- Runs a sanity overfit check
- Trains the custom CNN (25 epochs) and MobileNetV2 (8 + 18 epochs)
- Saves the best `.keras` checkpoint for each model
- Generates all figures in `figures/`

### 4. Convert to TF.js (for web deployment)

```bash
python convert_model.py
```

This auto-installs `tensorflowjs` if missing and converts `best_custom_cnn.keras` to the `model/` folder with uint8 quantization (~4× smaller file size).

### 5. Deploy to HuggingFace Spaces

```bash
# Create a new Gradio Space on huggingface.co, then push:
git init hf-space && cd hf-space
# Add app.py (Gradio interface) and your .keras model file
git add . && git commit -m "initial deploy"
git remote add hf https://huggingface.co/spaces/sid009991/fashion-classifier
git push hf main
```

A minimal `app.py` template:

```python
import gradio as gr
import tensorflow as tf
import numpy as np
from PIL import Image

model = tf.keras.models.load_model("best_mobilenetv2.keras")
classes = ["Topwear","Shoes","Bags","Bottomwear","Watches",
           "Innerwear","Jewellery","Eyewear","Fragrance","Sandal"]

def predict(img):
    img = img.resize((192, 192))
    x = np.array(img) / 255.0
    x_flip = np.fliplr(x)
    probs = (model.predict(x[None])[0] + model.predict(x_flip[None])[0]) / 2
    return {classes[i]: float(probs[i]) for i in range(10)}

gr.Interface(
    fn=predict,
    inputs=gr.Image(type="pil"),
    outputs=gr.Label(num_top_classes=3),
    title="Product Classifier",
    description="Upload a fashion product image to classify it into one of 10 sub-categories."
).launch()
```

## Key Findings

- **Both models exceed 98.5% accuracy** — near-ceiling performance on this 10-class dataset
- **MobileNetV2 leads on all metrics**, but the gap is <0.2% — the custom CNN is competitive despite training from scratch
- **Sandal is the hardest class**: ~18% of Sandal test samples are misclassified as Shoes in both models (visually similar silhouette)
- **Grad-CAM confirms meaningful attention**: heatmaps concentrate on product regions (frames for Eyewear, straps for Sandal, handles for Bags) rather than backgrounds

---

## Limitations

- Heavy class imbalance (Topwear = 42%, Sandal = 2.6%) — no focal loss or oversampling applied
- Controlled product shots only — real-world user photos may lower accuracy
- Custom CNN limited to 25 epochs due to compute budget
- Single 70/15/15 split; cross-validation would give variance estimates
