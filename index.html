What the notebook does

Pipeline in 5 steps

18 notebook cells, fully commented. Here's the order of operations.

01
Load & filter
Read styles.csv, resolve image paths, pick the 10 most common subCategory classes with ≥350 samples each.
02
Split & pipeline
Stratified 70/15/15 split, no leakage. tf.data pipeline with parallel JPEG decode, resize to 192×192, normalize to [0, 1].
03
Augment + MixUp
In-model layers (flip, rotate, zoom, contrast) plus batch MixUp at α=0.2. Disabled at inference automatically.
04
Train both models
Custom ResNet-SE (25 epochs, AdamW + cosine decay) and MobileNetV2 with two-phase fine-tuning (8 head + 18 fine-tune epochs).
05
Evaluate & visualize
Confusion matrices, ROC curves, Grad-CAM overlays, error analysis, and a side-by-side model comparison table. 13 figures saved.

Quick start

Run it yourself in 4 steps

No local GPU needed. The notebook is pre-configured for Kaggle paths — it also runs on Colab with a small path change.

Option A — Kaggle (recommended, no path changes needed)
# 1. Go to the Kaggle dataset page and click "New Notebook"
#    https://www.kaggle.com/datasets/paramaggarwal/fashion-product-images-small
# 2. Upload Product_classifier.ipynb from the GitHub repo
# 3. Enable GPU: Settings → Accelerator → GPU T4 x2
# 4. Run All — data paths are already correct
Option B — Google Colab (requires the Kaggle CLI)
# Install Kaggle CLI and download the dataset

Papers behind the notebook

Further reading

Every technique in the notebook has a paper worth understanding before your next project.

📄
Paper
Squeeze-and-Excitation Networks
Hu et al. (2018). The SE block used in every residual stage of the custom CNN. Recalibrates channel weights adaptively without major parameter cost.
arxiv.org · 2018 ↗
📄
Paper
MobileNetV2
Sandler et al. (2018). Inverted residuals and linear bottlenecks — the architecture behind the transfer learning comparison model.
arxiv.org · 2018 ↗
📄
Paper
MixUp Training
Zhang et al. (2018). Mixing pairs of training examples and labels. Used in the notebook at α=0.2 as a batch-level augmentation step.
arxiv.org · 2018 ↗
📄
Paper
Grad-CAM
Selvaraju et al. (2017). Gradient-weighted Class Activation Mapping. Used in Section 16 to visualize what the CNN focuses on for each prediction.
arxiv.org · 2017 ↗
📖
Guide
tf.data Performance Guide
Covers AUTOTUNE, prefetch, and parallel map — all patterns the notebook uses to keep the GPU fed during training.
tensorflow.org ↗
📖
Tutorial
Keras Grad-CAM Example
The official Keras tutorial the notebook's Grad-CAM implementation is based on. Includes the full GradientTape approach with working code.
keras.io ↗
